Как известно, лень — двигатель прогресса. Серьёзные разработчики не хотят тратить своё время на сохранение структур данных в текст, а потом скрупулёзное восстановление обратно в структуру данных. Они хотят, чтобы это можно было сделать автоматически, лучше одной строкой кода.

А ещё при обмене данными между различными сервисами приходится передавать очень разнородную информацию в большом объёме. Для её записи очень хорошо подходит структура, похожая на словарь _Python_. Почему не воспользоваться тем, что уже придумано? Сейчас многие приложения используют для обмена данными формат JSON, который может хранить структуры из языка _Python_ (и не только).

Так, можно отправить запрос погодному сервису и получить ответ в формате JSON, примерно такой:

```python
{
 "data": {
   "weatherByPoint": {
     "now": {
       "cloudiness": "CLEAR",
       "humidity": 98,
       "precType": "NO_TYPE",
       "precStrength": "ZERO",
       "pressure": 760,
       "temperature": 7,
       "fahrenheit": 44,
       "windSpeed": 3,
       "windDirection": "SOUTH_WEST"
     }
   }
 }
}
```

Сериализация — это процесс преобразования структур данных в какой-либо формат, чаще текстовый или байтовый для сохранения во внешнем хранилище. Десериализация — это обратный процесс, то есть преобразование строки обратно в структуру, которую понимает программа.

Модуль json как раз и предназначен для выполнения таких операций. Мы познакомимся с несколькими функциями из этого модуля, знание которых необходимо и достаточно для сериализации и десериализации структур данных _Python_.
