Хотя объекты JSON и похожи на объекты языка _Python_, они не идентичны. Начнём с того, что строки в JSON должны быть заключены в двойные кавычки. Никаких одинарных! Если вы создаёте файл программно, то беспокоиться незачем, правильные кавычки расставятся автоматически. О них следует помнить, если вы решите создать такой файл вручную.

Кроме того, не все типы данных из _Python_ имеют соответствие в JSON: словарь есть, список есть, строка и число — тоже, а вот кортежа нет, при записи такого объекта в JSON он будет преобразован в список. В таблице приведено соответствие типов данных _Python_ и JSON.

|   |   |
|---|---|
|**Python**|**JSON**|
|dict|object|
|list, tuple|array|
|str|string|
|int, float|number|
|True|true|
|False|false|
|None|null|

Кроме того, ключами словаря в JSON не могут быть числа; при преобразовании они будут заменены строками и заключены в двойные кавычки.

Итак, давайте посмотрим, как записать объект _Python_ в файл JSON. Для этого нам потребуется импортировать модуль json и взять из него метод dump().

Откроем файл, в который хотим сохранить объект, на запись, установим кодировку, если требуется.

```python
import json

dic = {'key': 'value',
       'key1': 'value1'}
with open('example.json', 'w') as out_file:
    json.dump(dic, out_file)
```

Готово, словарь записан в файл!

Метод json.dump() принимает обязательные аргументы — что записать и куда записать — и ещё несколько необязательных, некоторые из них мы рассмотрим, и ничего не возвращает, просто записывает объект в файл.

Есть ещё похожий метод json.dumps(), который преобразует объект в **строку** в формате json, которую и возвращает.

```python
s = json.dumps(dic)
print(s)
# {"key": "value", "key1": "value1"}
```

Среди необязательных аргументов обоих методов есть очень полезные, например такие:

ensure_ascii — если в записях есть кириллические символы, то они будут записаны в байтовом виде; чтобы вернуть написание в виде букв, установите значение этого параметра на False;

skipkeys — если ключи словаря не являются базовыми типами, то можно избежать ошибки TypeError, установив этот параметр в True, тогда они будут пропускаться при преобразовании;

sort_keys — если установить True, то в возвращаемом словаре ключи будут отсортированы.

## Примеры

```python
import json

dic = {'surname': 'Иванов',
       'name': 'Василий',
       1: 13,
       (False, 'age'): 5}
s = json.dumps(dic, ensure_ascii=False, skipkeys=True)
print(s)
# {"surname": "Иванов", "name": "Василий", "1": 13}
```

И с сортировкой ключей:

```python
dic = {'surname': 'Иванов',
       'name': 'Василий',
       'age': 13,}
s = json.dumps(dic, ensure_ascii=False,  sort_keys=True)
print(s)
# {"age": 13, "name": "Василий", "surname": "Иванов"}
```

Подробнее о методах и их аргументах можно прочитать в [документации](https://docs.python.org/3/library/json.html).

**Замечание**

При сдаче задач в тестирующую систему форматировать запись объектов в файл или строку не нужно.
