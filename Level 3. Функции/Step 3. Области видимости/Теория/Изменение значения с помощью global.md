Понимание принципа разделения областей видимости поможет нам при взаимодействии нашей функции с остальной программой. Пока мы могли только получать от функции результат её работы для использования в основной программе. А может ли функция всё же изменять значения переменных в глобальной области видимости? Оказывается, может, но делать это нужно очень аккуратно.

Часто над большими проектами работает сразу несколько программистов. Каждый из них может делать только часть — отдельную функцию, которая затем будет работать в общей программе. Если функция одного программиста изменит значения переменных основной программы, когда другие программисты об этом не знают и не ожидают изменений, то будет плохо, работа всей программы может измениться непредсказуемо.

Мы изучим, как можно изменять значения переменных в глобальной области видимости из функции (иногда без этого трудно обойтись), но пользоваться этими инструментами нужно с умом!

Рассмотрим такую задачу.

Функция count() при вызове должна печатать, который раз она вызвана. Если мы попробуем для счётчика количества вызовов завести локальную переменную n, то программа будет работать не так, как требуется:

```python
def count():
    n = 1
    print(n)
    n += 1


count()
count()
count()
# 1
# 1
# 1
```

Но всё верно: при каждом новом вызове создаётся новая локальная переменная n и ей присваивается значение 1. То, что значение затем увеличивается на 1, ничего не меняет, это значение мы нигде не используем, функция сразу после этого изменения перестаёт работать. Как же быть?

Выходит, переменная для хранения количества вызовов функции не должна быть локальной, она должна находиться в глобальной области видимости и хранить значение там. Вот так:

```python
n = 1


def count():
   print(n)
   n += 1
```

Но так наша функция вообще перестала работать. Строка, в которой мы увеличиваем значение n, говорит интерпретатору, что переменная локальная, но её значение нигде в функции не определено. А глобальную n изменить из функции нельзя. Что же делать? Сделать переменную глобальной!

Для таких случаев существует инструкция global, применять её надо так:

```python
n = 1


def count():
   global n
   print(n)
   n += 1


count()
count()
count()
# 1
# 2
# 3
```

Теперь всё отлично работает: переменная n стала глобальной, начальное значение задано в основной программе, а изменяем значение переменной — в функции.

Ещё раз повторим: использование глобальных переменных может иметь скрытые и неочевидные побочные эффекты, которые приводят к неожиданному поведению программы. Поэтому везде, где можно обойтись без global, нужно стараться обходиться.
