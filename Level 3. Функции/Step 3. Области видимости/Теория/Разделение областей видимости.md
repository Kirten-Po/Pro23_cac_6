Давайте представим себе ситуацию, в которой функции для выполнения своей задачи нужно узнать значение переменной, которая находится в основной программе. Например, это число π с заданной точностью. Можно ли из функции узнать это значение? Можно ли его использовать?

Нет ничего проще! Оказывается, в функции доступны все переменные, находящиеся в основной программе, в _глобальной области видимости_, как это называется. Поэтому функция может использовать эти значения для своих вычислений:

```python
def circle_length(radius):
    return 2 * pi * radius


pi = 3.14
print(circle_length(20))
# 125.60000000000001
```

Отлично! А если значение, необходимое в функции, — коллекция? Например, список? Можно его использовать? Можно:

```python
def distance():
    x1, y1, x2, y2 = points
    return ((x1 - x2) ** 2 + (y1 - y2) ** 2) ** 0.5


points = [1, 1, 4, 5]
print(distance())
# 5.0
```

Такой механизм работает только в одном направлении — из функции мы видим всё, что находится вокруг, в основной программе. Но наоборот не получается: из основной программы получить значения переменных, находящихся в функции, нельзя. Они находятся в другой области видимости — _локальной_. Это сделано для того, чтобы из основной программы кто-нибудь случайно не изменил, не «сломал» функцию.
