_Python_ — язык с динамической типизацией. Это значит, что интерпретатор сам определяет тип переменной по её значению, его не нужно объявлять заранее.

Однако при передаче в функцию аргументов интерпретатор не знает, какого они типа, пока функция не будет вызвана с конкретными значениями. Поэтому ваша IDE (среда программирования) не может проверять корректность производимых вами операций с этими аргументами и автоматически дополнять код.

Но это можно исправить, если использовать аннотации — указывать типы аргументов и возвращаемых значений. Аннотировать можно и переменные, но для функций это выглядит так:

```python
def slice_from_string(n: int, line: str) -> list:
    return list(line[::n])


print(slice_from_string(3, 'Hammerofforest'))
# ['H', 'm', 'o', 'o', 's']
```

Здесь после обозначения аргументов функции через двоеточие указан тип этого аргумента, а после определения функции через стрелочку — тип возвращаемого значения.

Если аргументы имеют одинаковый тип, то его записывают один раз. Для составных типов можно указать типы элементов:

```python
def powered_digits(n, power: int) -> dict[int: int]:
    return {x: x ** power for x in range(1, n)}


print(powered_digits(4, 5))
# {1: 1, 2: 32, 3: 243}
```

Аннотации предоставляют дополнительное удобство программисту в отслеживании кода, но, конечно, программа будет работать и без них.
