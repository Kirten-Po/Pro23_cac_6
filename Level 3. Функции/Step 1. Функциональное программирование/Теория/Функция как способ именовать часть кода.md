Довольно часто в программировании приходится выполнять одни и те же действия, возможно, немного меняя значения переменных, с которыми эти действия выполняются. Приходится заново писать код, который уже был реализован. Нельзя ли «сэкономить»?

Для того чтобы части кода присвоить имя, а затем обращаться к ней по этому имени, придуман механизм _функционального программирования_.

Мы уже многое знаем про функции, встроенные в _Python_. Например, это функции print(), input(), len(), max() и множество других.

Чтобы вызвать функцию — заставить её выполнить действия, — после её имени нужно поставить скобки.

Внутри скобок могут быть записаны значения, а могут и не быть: всё зависит от функции.

Функция может выполнять действия (в случае с print(), например, выводить на печать), а может возвращать значения (как len() возвращает длину строки).

Чтобы создать свою функцию, её нужно _объявить_ — описать в программе, а затем _вызвать_, и именно в таком порядке: нельзя вызвать то, что ещё не описано.

## Синтаксис объявления функции

После служебного слова def (от английского _definition_ — определение) записывается имя функции, затем круглые скобки, в которых могут быть записаны аргументы функции, а могут и нет, если аргументов нет, и двоеточие. После двоеточия следующая строка всегда начинается с отступа; затем идёт блок кода — собственно то, что должна делать функция.

```python
def <имя функции>(<возможно, аргументы>):
    <тело функции — блок кода>
```

Чтобы применить функцию, её нужно вызвать, то есть в программе записать имя функции с круглыми скобками после него. В скобках могут быть записаны значения аргументов, а могут и не быть, если функция вызывается без аргументов.

```python
<имя функции>(<возможно, аргументы>)
```

## Пример создания функции

Пусть в задаче в разных местах программы требуется выводить полосу клеточек из символов, вот такую:

-----------------------------------------
| * | * | * | * | * | * | * | * | * | * |
-----------------------------------------

Можно каждый раз копировать код, а можно написать функцию, содержащую этот код, и вызывать её в нужных местах.

```python
def stripe():
    print('-' * 41)
    print('| * ' * 10 + '|')
    print('-' * 41)


stripe()  # вызов функции
```

**PEP8**

По правилам PEP8 определение функции отделяется от остального кода двумя пустыми строками. Если в коде определяется несколько функций, то между ними нужно также оставлять две пустые строки.

```python
def first():
    ...


def second():
    ...


def third():
    ...


first()
second()
third()
```

Здесь многоточие — это пустая инструкция, которая ставится там, где обязательно должен быть какой-то код, например после двоеточия в предыдущей строке.

## Именование функций

В Python существуют правила именования функций, они во многом совпадают с именованием переменных. Вспомните:

- функции принято именовать латинскими буквами в нижнем регистре (стиль _snake_case_);
- кроме букв, имя функции может содержать только цифры (но не на первом месте!) и символ нижнего подчёркивания (_);
- слова в имени функции принято разделять символом нижнего подчёркивания.

Ну и ещё одно общее правило для имён переменных и функций: называйте их так, чтобы было понятно (в первую очередь вам самим, но и тем, кто будет читать ваш код), что содержится в переменной или что делает функция.

## Пример функции для расчёта значений

Число π можно приблизительно рассчитать, если разделить 22 на 7. Напишем функцию, которая рассчитывает и выводит приближённое значение числа π.

```python
def pi_number():
    print(22 / 7)


pi_number()  # 3.142857142857143
```
