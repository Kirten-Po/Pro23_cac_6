В функции именованные аргументы всегда и **обязательно** должны идти после позиционных. А вот порядок самих именованных аргументов не важен, если они при вызове записываются по именам, а не по положению.

Напишем функцию, которая собирает мелодию из списка переданных нот и добавляет автора мелодии. Вот так:

```python
def make_melody(args, name):
    return f'Author: {name}\nMelody: {"-".join(args)}'


print(make_melody(['La', 'la', 'fa'], 'Kate'))
```

Или так:

```python
print(make_melody(['La', 'la', 'fa'], name='Kate'))
```

В первом примере именованный аргумент при вызове не назван, так как всего передаётся два аргумента и по положению имени понятно, что это именно оно, а не список нот.

В случае с произвольным числом аргументов так сделать не получится, поскольку неизвестно заранее, где заканчиваются позиционные аргументы — ноты — и начинается имя. Поэтому вызов функции может быть только такой:

```python
def make_melody(*args, name):
    return f'Author: {name}\nMelody: {"-".join(args)}'


print(make_melody('La', 'la', 'fa', name='Kate'))
```

Количество именованных аргументов может быть любым, порядок тоже, главное, чтобы они при вызове были названы и стояли после позиционных аргументов. Например, так:

```python
def make_melody(*args, name, age):
    return f'Author: {name}\nAge: {age}\nMelody: {"-".join(args)}'


print(make_melody('La', 'la', 'fa', name='Kate', age=12))
print(make_melody('La', 'la', 'fa', age=12, name='Kate'))
```
