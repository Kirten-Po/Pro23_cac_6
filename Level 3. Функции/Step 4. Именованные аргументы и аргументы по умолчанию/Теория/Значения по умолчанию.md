В примерах встроенных функций, которые мы рассмотрели, все именованные аргументы имели значения по умолчанию, то есть такие, которые должны быть подставлены в функцию при вызове, если они не заданы явно.

Например, аргументы sep и end функции print() имеют значения по умолчанию «пробел» и «перевод строки» соответственно.

Функция int() будет по умолчанию считать, что переданное ей число для преобразования записано в десятичной системе счисления.

У метода строк replace() также есть именованный аргумент со значением по умолчанию: количество замен, которые нужно сделать.

```python
s.replace('old', 'new', 3)
```

Если это значение не указать, то метод сделает все возможные замены. Это удобно, поскольку чаще всего функции применяются именно с такими значениями именованных аргументов. В то же время поведение функции можно изменить, задавая другие значения этих аргументов. Это делает применение функций удобным и гибким одновременно.

Рассмотрим пример решения задачи, в которой применение аргументов со значениями по умолчанию удобно и оправданно.

## Пример решения задачи

Пусть нужно написать функцию **spaceship()** для подготовки космического корабля к старту. Функция принимает произвольное число грузов и именованные аргументы fuel, crew и maps (топливо, экипаж и карты) со значениями по умолчанию соответственно False, 0 и ′Vega′.

Если топливо заправлено (True), экипаж состоит как минимум из 4 человек, а карты предназначены для полёта к звезде (то есть не являются пустой строкой), то можно стартовать! В этом случае функция возвращает суммарную длину и количество всех аргументов-грузов.

Если стартовать нельзя, то нужно вернуть значения именованных аргументов в порядке перечисления в условии.

_Примеры вызова функции:_

|   |   |
|---|---|
|```python<br>print(spaceship(23, 42, 17, crew=5, maps=''))<br>```|```python<br>(False, 5, '')<br>```|
|```python<br>print(spaceship(23, 42, 17, fuel=True, maps=''))<br>```|```python<br>(True, 0, '')<br>```|
|```python<br>print(spaceship(23, 42, 17, crew=4, fuel=True))<br>```|```python<br>(82, 3)<br>```|

Решение

```python
def spaceship(*cargo, fuel=False, crew=0, maps='Vega'):
   if fuel and crew == 4 and maps:
       return sum(cargo), len(cargo)
   return fuel, crew, maps
```

## Изменяемые типы как значения по умолчанию

Если в качестве значения по умолчанию нужно использовать какой-либо изменяемый тип данных — список, множество, словарь, — то надо быть осторожным, ведь в функцию вообще передаётся не переменная, а её значение. Поэтому поведение функции может стать для вас неожиданным.

Пусть в функции нужно дописать переданный позиционный аргумент-ноту в список нот, по умолчанию пустой, и вернуть его.

**Вот так делать неправильно:**

```python
def make_melody(nota, notes=[]):
    notes += [nota]
    return notes


print(make_melody('La'))  # ['La']
print(make_melody('la'))  # ['La', 'la']
print(make_melody('fa'))  # ['La', 'la', 'fa']
```

Мы ожидали, что при каждом новом вызове будет создаваться новый список, но этого не происходит, объект-список создан один раз в процессе выполнения программы и каждый раз дополняется.

Правильно поступить так: в качестве значения по умолчанию записать None, а в программе проверить это значение и поступать по-разному в зависимости от него.

**Так правильно:**

```python
def make_melody(nota, notes=None):
    if not notes:
        return [nota]
    notes += [nota]
    return notes


print(make_melody('La'))  # ['La']
print(make_melody('la'))  # ['la']
print(make_melody('fa'))  # ['fa']
print(make_melody('do', notes=['La', 'la', 'fa']))
# ['La', 'la', 'fa', 'do']
```
