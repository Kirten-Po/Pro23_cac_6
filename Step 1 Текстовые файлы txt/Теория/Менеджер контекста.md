Но есть ещё один способ работы с файлами, который не требует закрытия, — это менеджер контекста with. Синтаксис его такой:

```python
with open(<имя файла, параметры>) as f:
    ...
```

Многоточие — это пустая инструкция, её ставят там, где обязательно должен быть какой-то код, но ничего делать не надо.

Как обычно, в _Python_ после двоеточия блок кода начинается с отступа. Все операции в этом блоке кода будут происходить при открытом файле. Как только вы напишете команду без отступа, файл будет закрыт.

```python
with open('example.txt') as f:
    print(f.read())
print('OK')
```

В менеджере контекста можно сразу открывать несколько файлов — например, один для чтения, другой для записи через запятую:

```python
with open('example.txt') as fin_file, open('result.txt', 'w') as out_file:
    ...
```

Следите, чтобы переменные, с которыми сопоставляются файловые объекты, имели разные имена.

## Пример решения задачи

В файле vocabulary.txt записаны слова и через дефис, окружённый пробелами, их перевод на английский. В файле requests.txt записаны слова для перевода. В файл answers.txt нужно для каждого слова-запроса записать его перевод, если оно было в файле, а если не было, то оставить пустую строку.

Пример файла _vocabulary.txt_:

```python
кот - cat
собака - dog
слон - elephant
верблюд - camel
```

Пример файла _requests.txt_:

```python
верблюд
синичка
слон
кот
```

_Решение_

```python
with (open('vocabulary.txt', encoding='utf-8') as voc,
      open('requests.txt', encoding='UTF8') as req,
      open('answers.txt', 'w') as ans):
    dic = {}
    for line in voc:
        k, v = line.rstrip().split(' - ')
        dic[k] = v
    for question in req:
        if question.rstrip() in dic:
            ans.write(dic[question.rstrip()] + '\n')
        else:
            ans.write('\n')
```

Файл answers.txt:

```python
camel

elephant
cat
```
